4.SOLID Principles (OOP Core)

S — Single Responsibility Principle
Using custom hooks and small reusable components.

O — Open/Closed Principle
Add new behavior without modifying existing code

L — Liskov Substitution Principle
Components should be replaceable without breaking app

I — Interface Segregation Principle
Don’t pass unnecessary props/methods

D — Dependency Inversion Principle
Keeps Abstraction which is easier for dependency injection
